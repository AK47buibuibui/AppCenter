# 软件使用说明

> [视频介绍地址](https://www.bilibili.com/video/av51859853)

## 订阅源的获取

RSS为人诟病的一点就在于订阅源需要自己去找，不过发展至今，已经有了非常多的聚合RSS订阅源，我在这里推荐两种订阅源的获取方式：

1. 国内用户可以访问 [RSS Hub](https://docs.rsshub.app)，里面包含了非常多的订阅源，但问题在于这些订阅源的访问速度不一，觉得卡的话可以考虑自己部署相关服务包。
2. 软件提供了便捷的订阅源搜索，你可以在建立新频道时，通过输入订阅源地址或关键词的方式直接搜索订阅源。订阅源搜索服务基于[Feedly](https://feedly.com)。

## 订阅源的管理

软件提供了上下层的管理结构。

所谓下层，即提供了简单的二级分类——`类-频道` 的方式。

![others_rss_sidebar.png](https://storage.live.com/items/51816931BAB0F7A8!12766?authkey=AO7QXpgYo7-5DUU)

左侧是分类，也即为你的频道分组，你可以自选图标。

右侧是当前分类下的频道列表，上方可以进行标题搜索，旁边的蓝色圆按钮就是添加频道的按钮了，点击之后可以通过链接或关键词的方式添加新的订阅源。

而上层，则是建立在你已有订阅源的基础上，允许你对订阅源进行整合，并让所选订阅源呈现在同一页面上，该层级被称为“自定义页面”

:::tip
在输入订阅源的时候，需要先按放大镜按钮或者回车来进行订阅源的**试连接**，连接不成功是无法建立频道的。
:::

你可以通过鼠标右键点击对应的列表项弹出控制菜单（触屏下是长按）：

![others_rss_contextmenu.png](https://storage.live.com/items/51816931BAB0F7A8!12767?authkey=AO7QXpgYo7-5DUU)

你也可以直接拖动列表项来对其进行排序。

## 自定义页面

![others_rss_addpage.png](https://storage.live.com/items/51816931BAB0F7A8!12768?authkey=AO7QXpgYo7-5DUU)

你可以点击“页面”旁的加号按钮来添加新的自定义页面，界面如上。

如果你需要修改或将所选页面设置为“首页”，请右键单击（触屏为长按）对应的页面项。

在左侧，你需要勾选你想要整合的订阅源，这些订阅源是你事先就已经建立好的。

**这里再次说明，你必须已经有订阅源才能使用自定义页面的功能。**

右侧的规则添加需要进行一些说明：

1. 文本类规则。
   目前有 `需要的标题或内容` 以及 `不需要的标题和内容`，它的目的是为了筛选出符合你要求的文章，输入的内容为正则表达式。但是不用担心，如果你只是需要一些关键词的过滤，那么以竖线分割即可。
   举个例子，我想过滤出所有文章中包含`Windows`，`UWP`字眼的文章，那么我可以直接输入`Windows|UWP`。如你所见，竖线分割即可。如果你会一些更为复杂的正则表达式，那你也可以自行编写。
2. 数值类规则。
   目前有`总体最大条目数`和`单个最大条目数`。它们的目的就是为了限制条目的数量，所以你必须要输入数字（而且是正整数）。
   所谓`总体最大条目数`，顾名思义，就是将所有所选订阅源的文章整合后，按时间排序，剔除超过指定数量的文章。
   而`单个最大条目数`和总体类似，它用来限制每一个订阅源最多显示多少文章。

## 文章操作

![others_rss_mainpage.png](https://storage.live.com/items/51816931BAB0F7A8!12769?authkey=AO7QXpgYo7-5DUU)

在频道详情页面中，你可以看到当前订阅源所提供的最新文章，软件会以瀑布流的形式展示文章列表，每一篇文章都是一个独立的卡片，卡片提供两个简单操作：

- 通过网页打开
- 分享

:::tip
文章详情的排版提供了三种方式，即`瀑布流`(图中显示的那种),`卡片式`和`列表式`，以供不同喜好的用户自行选择。
:::

直接点击卡片即可进入对应文章详情。

![others_rss_articledetail.png](https://storage.live.com/items/51816931BAB0F7A8!12770?authkey=AO7QXpgYo7-5DUU)

文章详情页面中，位于标题左边的按钮的功能是返回文章列表。

### 已读/未读

每一篇文章都有标识已读/未读的阅读状态，该状态不能手动更改。用户点击文章则该文章状态就会被设置为已读。你也可以点击“全部已读”按钮将该频道内所有文章都设置为已读。
通过“仅显示未读文章”的切换按钮，可以提高你刷资讯的效率。

### 全文操作

在文章右侧，提供了一些快捷操作按键。

1. 蓝色的菜单按钮，用于控制侧边栏的开启与关闭
2. 网络按钮，点击可以打开该文章对应的网页
3. 标签按钮，将该文章加入待读列表
4. 星星按钮，将该文章加入收藏列表
5. 云形按钮，获取文章全文。有些订阅源只提供了简单的摘要，通过这个按钮可以对原网页进行解析，并将其转换为可视文本以供阅读（借助Readability实现）
6. 分享按钮，打开分享菜单，你可以选取合适的方式对当前文章进行分享
7. 底部的More按钮，这里隐藏了一些不常用的功能：
   1. 内容重载。将当前页面的内容初始化。
   2. 全文翻译。借助百度翻译实现，需要提供百度翻译平台的Key和AppId（自行申请）。不过不得不说，翻译效果不好，很影响排版。
   3. 朗读。该功能借助Windows自身语音功能实现，为有特殊需要的人群服务，你可以在设置中设置朗读者的性别和语速。
8. 在文章内，当你点击图片时，会弹出图片框，你可以复制或保存图片。

### 划词操作

针对阅读过程，软件也附加了一些功能，这些功能体现在`划词菜单`中。

![others_rss_selecttextmenu.png](https://storage.live.com/items/51816931BAB0F7A8!12771?authkey=AO7QXpgYo7-5DUU)

当你在阅读时选中文本，划词菜单会自动弹出，并询问你打算做什么事情：

1. 翻译。全文翻译不顶用，划词翻译还是挺好使的。尤其是在阅读不同语言的文本时，省去了打开词典的步骤。只不过该功能同样需要百度翻译的`AppId`和`Key`。
2. 标记文本。高亮标记选中文本，标记后，如果你分享该文章，则标记也会被带上。但要注意，标记区域不能重复，否则会报错（所谓标记，就是给选中文本套了个\<mark\>标签，所以不能嵌套）
3. 在网页中搜索。可以直接生成搜索链接，并打开默认浏览器进行搜索。搜索引擎可在软件的设置->基础设置中查看，目前支持`Google`,`Bing`和`百度`。
4. 分享。分享选中文本。
5. 朗读。朗读选中文本。

:::tip
稍后阅读列表和收藏列表的入口在软件左下角，和设置按钮并列。
:::

:::warning
软件所提供的稍后阅读或收藏功能是很简陋的，事实上我也不准备去 **“完善”** 。提供这两个功能的主要目的在于帮助你把喜欢的文章固定下来，真正要做摘抄也好，永久保留也好，建议使用OneNote或者印象笔记一类的专门化应用。
:::

## 特殊功能

### 时间线支持

软件提供了对Windows10特色功能`Timeline`的支持。你浏览过的文章都会以自适应卡片的形式出现在Timeline中，这样即使你切换了设备，同样可以通过时间线来快速回阅。这也弥补了软件本身没有历史记录的不足。

### OneDrive同步

在初次运行软件时，你可以绑定OneDrive（不想的话也可以使用本地数据）。绑定之后，你的订阅、稍后阅读、收藏列表还有推送列表都会在云端备份。每当你进行列表的改动，都会同步到云端，你的其它的设备也可以进行同步。

:::tip
实时同步有一个前提，即只会同步登录了同一微软账户的设备。如果你绑定了相同的OneDrive，但是登录的微软账户不同，那么这两个设备是不能进行实时同步的。这个时候你就需要在设置中进行手动同步。
:::

### 主动缓存订阅源

软件允许主动缓存订阅源（默认关闭，在数据设置中打开）。开启后，每隔15分钟软件会刷新一次当前已有的订阅源，在软件打开时会优先读取缓存的内容。
这样既保证了时效性，也会极大提高载入速度。
但如果某个订阅源难以在有效时间内获取数据，则该订阅源会被放弃缓存。

### 语音朗读

通过Windows提供的Speech工具，可以朗读文章或选定文本，功能如何使用因人而异。我看国外文章时，会经常用到划词翻译和朗读的功能，了解某个单词的含义及发音。

虽然这两者都不见得标准，但是对于辅助理解文章内容还是有不小的帮助的。

### 网页翻译

使用百度翻译API进行网页翻译，不保证准确率和翻译程度，用它只是因为注册简单且有很高的免费额度。

使用该功能需要你自行在百度翻译中心注册，很简单，注册之后可以在个人主页的底部找到自己的`AppId`和`Key`

### 订阅推送

你可以自主选择推送哪些订阅源的消息，方法就是在右键点击对应的订阅源（触屏上是长按），在弹出菜单中选择订阅通知即可。

软件的后台每隔15分钟检查一次订阅源的更新状态，如果有新的文章会进行推送通知。

即便你关闭了软件，也不会影响到后台程序的运行，所以不用担心。

:::tip
UWP对于后台的唤醒有着较为严格的限制，最低15分钟一次，且单次最长运行时间为25秒左右。我已尽力优化，但仍有可能存在网络问题导致的尚未获取数据，而后台就被强制结束的情况。
:::

### OPML导入及导出

如果你在接触到本软件之前就已经有了自己惯用的订阅软件或服务，相信你已经有了自己的一套订阅列表。如果你想迁移到本软件那也很简单。

成熟的RSS订阅服务都会提供OPML格式用于进行订阅列表的导入及导出。你可以将你当前的订阅源列表导出为OPML格式，并在软件的设置内导入OPML即可无缝接入到软件之中。

当然，由于OPML并不包含分类的图标属性，所以软件在导入时会默认提供分类的基础图标（RSS图标），如果你需要，可以自己手动调整。
